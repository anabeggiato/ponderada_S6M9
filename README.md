# ponderada_S6M10

&emsp; Este repositório é a entrega do Grupo TechSwift composto por Lucas Nepomuceno, Gabriel Martins, Ana Beggiato e Cecília Lima. Nessa atividade de programação do Módulo 10 de Engenharia da Computação do Inteli desenvolvemos como entregável um aplicativo de programas de TV. É importante notar que escolhemos a categoria "ir além" e desenvolvemos a ponderada com as limitações de uma única view impostas pelo professor. Neste documento, você encontrará o raciocínio do grupo para a entrega:

&emsp; Primeiro, criamos o repositório no GitHub e a estrutura de pastas do projeto. Nesse processo, tivemos algumas dificuldades de adaptação no Mac, por exemplo, não sabíamos renomear arquivos. Então dentro de models, colamos o código fornecido pelo professor. Tivemos muitas dúvidas do que colar realmente dos esqueletos já que decidimos ir além... Inicialmente, porém, colamos tudo. Começamos, então, pelo NavigationLink, entendendo sua implementação. Nesse ponto, tivemos um problema no que tange à entender a sintaxe desse componente, então deixamos ele de lado no meio tempo. Partimos para fazer a ShowCard, primeiro puxamos do models/programa dinamicamente os dados e depois fizemos a alteração de colocar as cores dinamicamente com a função getColor(). Nesse ponto, tudo fluiu mais calmamente porque aprendemos finalmente a dinâmica da linguagem.
 
&emsp; Problemas só foram aparecer de novo em InfoBadge, quando tivemos algumas dificuldades para tornar o componente reutilizável. Claro, ele era reutilizável entre programas, mas não entre informações. Isso significa que reutilizávamos ele para Naruto e Avatar, mas não para "episódios" e "temporadas". Aqui, começamos um trabalho assíncrono. Metade do grupo ficou focada na continuação do ProgramaDetails, enquanto a outra metade, planejou e implementou um rascunho de como tornar o componente reutilizável. Isso funcionou e mudamos bastante a estrutura do botão, inclusive paramos de receber programa para receber apenas aquilo que era importante para o InfoBadge, como o tipo, o valor e o emoji. Neste componente, apenas o _stroke_ continuou a ser uma dúvida para o grupo até o final do tempo.

&emsp; Então, chegamos no final: inserir o botão e testar a usabilidade. A documentação no SwiftUI para o botão é muito precária e por isso utilizamos esse artigo do Medium: https://21zerixpm.medium.com/buttons-and-controls-in-swiftui-building-interactive-uis-8a450d2aaaeb. Em relação a usabilidade, fizemos apenas alguns ajustes de altura e largura para o badge de "Anime", "Série" ou "Desenho. Assim, finalizamos a ponderada.
